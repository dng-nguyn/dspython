\section{1. Introduction}\label{introduction}

\subsection{1.1 Problem Statement}\label{problem-statement}

Tourism is one of Vietnam's most important economic sectors,
contributing significantly to GDP and creating millions of jobs. In
recent years, Vietnam has emerged as an attractive destination in
Southeast Asia, drawing large numbers of international tourists from
diverse countries.

However, international tourist arrivals are influenced by complex
factors: global economic conditions, visa policies, exchange rates,
regional competition, and exogenous shocks such as the COVID-19 pandemic
(2020-2021). Understanding trends, seasonal patterns, and relationships
between source markets is essential for effective tourism strategy
planning.

A critical challenge in this dataset is \textbf{uneven country coverage
across years}: only 12 countries reported data during 2009-2011,
compared to 30+ from 2012 onward. This means raw aggregate totals are
partially inflated by reporting coverage changes rather than actual
growth. This report addresses this issue explicitly.

\subsection{1.2 Objectives}\label{objectives}

\textbf{Analytical objectives:} - Analyze overall trends in
international arrivals to Vietnam (2008-2026) - Identify top source
countries and their changing rankings - Detect seasonal patterns in
quarterly arrivals - Analyze correlations between source markets -
Assess the impact of COVID-19 on Vietnam's tourism sector - Evaluate
average annual growth rates, accounting for coverage gaps

\textbf{Predictive objectives:} - Build machine learning models to
predict quarterly arrivals - Compare Linear Regression, Random Forest,
XGBoost, SARIMA, and Chronos-T5 - Tune hyperparameters to improve model
accuracy - Forecast arrivals for the next 4 quarters with confidence
intervals - Evaluate the CIR\# stochastic model (from academic
literature) and explain why it does not fit this dataset

\subsection{2.2 Data Structure and
Challenges}\label{data-structure-and-challenges}

The \texttt{.xls} files are not real Excel binaries --- they are HTML
files with an \texttt{.xls} extension, a common technique for generating
``Excel'' files from web applications. The HTML structure presents
several challenges:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\item
  \textbf{Text nodes outside \texttt{\textless{}td\textgreater{}} tags:}
  Numeric values appear as text between empty
  \texttt{\textless{}td\textgreater{}} elements rather than inside them.
  Standard parsers like \texttt{pd.read\_html()} return all NaN values.
  A custom \texttt{lxml}-based parser was built to extract data from
  text nodes.
\item
  \textbf{Inconsistent year coverage:} Q3/Q4 files include 2008 data;
  Q1/Q2 start from 2009. The year 2021 is absent from all files
  (COVID-19 lockdown).
\item
  \textbf{Vietnamese number formatting:} Dots serve as thousands
  separators (e.g., \texttt{104.520} = 104,520), not decimal points.
\item
  \textbf{Regional aggregates mixed with countries:} The 2008 file
  includes entries like ``Chau A'' (Asia), ``Chau Au'' (Europe), and
  ``Other markets'' alongside individual countries. These were excluded
  from analysis.
\end{enumerate}

\subsection{2.3 Data Parsing and
Merging}\label{data-parsing-and-merging}

A custom HTML parser using \texttt{lxml} was built: - Parse the DOM tree
and extract year labels from the header row (columns 2+) - For each data
row, extract country name from \texttt{td{[}0{]}.tail} and values from
\texttt{td{[}1:{]}.tail} - Convert Vietnamese-formatted numbers by
removing dot separators - Merge all 4 files into a single DataFrame with
columns: \texttt{country}, \texttt{year}, \texttt{quarter},
\texttt{arrivals}

\textbf{Verification:} ``USA'' for 2009 Q1 = 104,520 --- confirmed
against the provided sample.

After parsing: \textbf{1,622 records}, \textbf{32 individual countries},
years 2008-2026.

\section{5. Model Building}\label{model-building}

This section describes each forecasting model applied to aggregate
quarterly arrivals (all countries summed). Every model uses the same
feature set: year, quarter number, time index, lag\_1 (previous
quarter), lag\_4 (same quarter last year), and rolling\_mean\_4
(4-quarter moving average).

Models are evaluated on the test set (2024 Q1 -- 2026 Q2, 10 quarters)
using four metrics: - \textbf{MAE} (Mean Absolute Error): average
absolute deviation from actual values - \textbf{RMSE} (Root Mean Squared
Error): penalizes large errors more heavily - \textbf{MAPE} (Mean
Absolute Percentage Error): scale-independent accuracy measure -
\textbf{R²} (Coefficient of Determination): proportion of variance
explained; 1.0 = perfect, 0 = predicting the mean, negative = worse than
the mean

\subsection{5.1 Linear Regression}\label{linear-regression}

\textbf{Algorithm overview:}

Linear Regression is the simplest supervised learning model. It assumes
a linear relationship between the input features x and the target
variable y:

\[y = \mathbf{w}^T \mathbf{x} + b\]

where \textbf{w} is the weight vector and \emph{b} is the bias term. The
model finds \textbf{w} and \emph{b} by minimizing the sum of squared
residuals (Ordinary Least Squares):

\[\min_{\mathbf{w}, b} \sum_{i=1}^{n} (y_i - \mathbf{w}^T \mathbf{x}_i - b)^2\]

This has a closed-form solution, making it extremely fast to train.

\textbf{Why it works here:} Despite its simplicity, Linear Regression
effectively captures the overall upward trend in Vietnam's tourism
arrivals. The \texttt{time\_idx} feature (year + quarter/4) provides a
strong linear signal. With only 51 training samples, the model's
simplicity is actually an advantage --- it does not overfit.

\textbf{Limitations:} Linear Regression cannot capture nonlinear
patterns, seasonal cycles, or the COVID-19 structural break. It assumes
the future is a linear extrapolation of the past.

\subsection{5.2 Random Forest
Regression}\label{random-forest-regression}

\textbf{Algorithm overview:}

Random Forest is an ensemble method that combines hundreds of decision
trees. Each tree is trained on a random bootstrap sample of the data,
and at each split node, only a random subset of features is considered.
The final prediction is the average of all tree predictions:

\[\hat{y} = \frac{1}{T} \sum_{t=1}^{T} h_t(\mathbf{x})\]

where \(h_t\) is the \(t\)-th tree and \(T\) is the total number of
trees.

The key insight is that individual trees are high-variance (they
overfit), but averaging many uncorrelated trees reduces variance while
preserving the ability to capture nonlinear patterns. This is known as
\textbf{bagging} (Bootstrap Aggregating).

\textbf{Key hyperparameters:} - \texttt{n\_estimators}: Number of trees
(200 in this study) - \texttt{max\_depth}: Maximum depth of each tree
(10 --- prevents overfitting on small data) -
\texttt{min\_samples\_split}: Minimum samples required to split a node

\textbf{Why it is challenging here:} With only 51 training samples,
Random Forest has limited data to build diverse trees. The random
subspace mechanism loses information when feature count is small.
Overfitting is a real risk.

\subsection{5.3 XGBoost Regressor}\label{xgboost-regressor}

\textbf{Algorithm overview:}

XGBoost (Extreme Gradient Boosting) builds trees \textbf{sequentially}
rather than in parallel. Each new tree focuses on correcting the errors
of the previous ensemble. The prediction after \(K\) trees is:

\[\hat{y}_i = \sum_{k=1}^{K} f_k(\mathbf{x}_i)\]

At each step \(k\), a new tree \(f_k\) is added to minimize a
regularized objective:

\[\mathcal{L}^{(k)} = \sum_{i=1}^{n} \ell(y_i, \hat{y}_i^{(k-1)} + f_k(\mathbf{x}_i)) + \Omega(f_k)\]

where \(\ell\) is the loss function (squared error for regression) and
\(\Omega(f_k)\) penalizes tree complexity (number of leaves and leaf
weights).

XGBoost uses \textbf{second-order Taylor expansion} of the loss function
for faster and more accurate optimization, plus \textbf{column
subsampling} (like Random Forest) to reduce overfitting.

\textbf{Key hyperparameters:} - \texttt{n\_estimators}: Number of
boosting rounds (200) - \texttt{max\_depth}: Tree depth (6 --- deeper
trees capture more interactions) - \texttt{learning\_rate}: Shrinkage
factor applied to each tree (0.1 --- smaller values need more trees but
generalize better) - \texttt{subsample}: Fraction of data used per tree
(1.0 here) - \texttt{colsample\_bytree}: Fraction of features used per
tree (1.0 here)

\textbf{Why it struggles here:} XGBoost's sequential correction
mechanism amplifies noise when the training set is small. With 51
samples and 6 features, the model memorizes training patterns that do
not generalize.

\subsection{5.4 SARIMA (Seasonal AutoRegressive Integrated Moving
Average)}\label{sarima-seasonal-autoregressive-integrated-moving-average}

\textbf{Algorithm overview:}

SARIMA is a classical statistical model for time series forecasting. It
decomposes the series into four components:

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  \textbf{AR (AutoRegressive):} The current value depends on its own
  past values
\item
  \textbf{I (Integrated):} Differencing to achieve stationarity
  (removing trend)
\item
  \textbf{MA (Moving Average):} The current value depends on past
  forecast errors
\item
  \textbf{S (Seasonal):} The same three components repeated at seasonal
  lag \(s\) (here \(s=4\) for quarterly data)
\end{enumerate}

The full SARIMA\((p,d,q)(P,D,Q)_s\) model is:

\[\Phi_P(B^s)\phi_p(B)(1-B^s)^D(1-B)^d y_t = \Theta_Q(B^s)\theta_q(B)\varepsilon_t\]

In this study: SARIMA\((1,1,1)(1,1,1)_4\) --- one autoregressive and one
moving average term at both the regular and seasonal level, with
first-order differencing at both levels.

\textbf{Key parameters:} - \((p,d,q) = (1,1,1)\): Non-seasonal AR order,
differencing order, MA order - \((P,D,Q)_s = (1,1,1)_4\): Seasonal AR,
differencing, MA orders with period 4

\textbf{Why it fails here:} SARIMA assumes the time series is stationary
after differencing. The 2021 data gap breaks the seasonal continuity ---
the model sees 2020 Q4 followed directly by 2022 Q1, creating an
artificial jump that distorts the estimated seasonal pattern.
Additionally, SARIMA cannot model the strong post-COVID growth trend,
which is a structural break rather than a continuation of pre-2020
dynamics.

\subsection{5.5 Chronos-T5 (Foundation Model for Time
Series)}\label{chronos-t5-foundation-model-for-time-series}

\textbf{Algorithm overview:}

Chronos is a family of pretrained foundation models for time series
developed by Amazon (Ansari et al., 2024). Unlike the other models in
this study, Chronos does not learn from our 51 training samples --- it
was pretrained on \textbf{millions of time series} from diverse domains
(retail, finance, weather, web traffic, etc.) using a Transformer
architecture (T5).

Chronos works by \textbf{tokenizing} time series values into discrete
bins and training an autoregressive language model to predict the next
token. At inference time, it generates multiple possible futures via
sampling, producing a \textbf{probabilistic forecast} (a distribution
over possible outcomes, not just a point estimate).

The key innovation is \textbf{zero-shot forecasting}: Chronos can
predict any new time series without task-specific training. You simply
pass the historical observations as context, and the model generates
forecasts.

\subsection{5.6 CIR\# --- Stochastic Differential Equation Model
(Considered but Not
Adopted)}\label{cir-stochastic-differential-equation-model-considered-but-not-adopted}

The CIR\# (Cox-Ingersoll-Ross Sharp) model from Clements et al.~(2023)
extends the classic CIR stochastic differential equation for tourism
forecasting with disrupted data. It achieved MAPE 1.18\% on Italian
monthly tourism data --- a 70\% error reduction vs SARIMA and
Holt-Winters.

\textbf{Algorithm overview:}

The classic CIR process models mean-reverting stochastic dynamics:

\[dr(t) = \kappa(\theta - r(t))dt + \sigma\sqrt{r(t)}dW(t)\]

where \(\kappa\) is mean-reversion speed, \(\theta\) is the long-run
mean, \(\sigma\) is volatility, and \(dW(t)\) is a Brownian motion
increment.

The CIR\# extension replaces Brownian motion with \textbf{ARIMA-filtered
residuals}: instead of generating random noise, the model uses the
actual forecast errors from an ARIMA model fitted to a rolling window.
This gives ``exact trajectory'' fitted values rather than Monte Carlo
averages. The data is also partitioned into subsamples to capture
variance changes around structural breaks (e.g., COVID-19).

\textbf{Why CIR\# does not fit this dataset:}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  \textbf{Insufficient observations:} Only 55 quarterly data points (vs
  288 monthly in the Italian study). The rolling window of M=8 consumes
  15\% of data.
\item
  \textbf{Mean-reversion assumption violated:} The CIR SDE pulls toward
  a long-run mean \(\theta\). Vietnam's tourism has a strong upward
  trend that contradicts mean-reversion.
\item
  \textbf{COVID shock magnitude:} An 80\% drop followed by full recovery
  causes the \(\sqrt{r}\) volatility term to amplify noise in
  log-returns, producing wild oscillations.
\item
  \textbf{Quarterly vs monthly granularity:} The original paper's
  success relied on monthly data providing more observations per window
  and better seasonal capture.
\end{enumerate}

CIR\# achieved MAPE \textasciitilde183\% on this data --- far worse than
all other models. This is an honest negative result: \textbf{not every
state-of-the-art model fits every dataset}. CIR\# is designed for
stationary, mean-reverting processes (interest rates, short-term
economic indicators), not for trending series with structural breaks.

\textbf{Result:} Optimization provided marginal improvement for RF but

This confirms that with 51 samples, simpler models are preferred.

\subsection{6.3 External Features (Exchange Rates + Visa
Policy)}\label{external-features-exchange-rates-visa-policy}

Exchange rates (VND vs KRW, CNY, USD, JPY, TWD, MYR) were fetched from
Yahoo Finance. Visa policy indicators were manually encoded: -
\texttt{visa\_evisa}: E-visa available (2017+) -
\texttt{visa\_evisa\_full}: E-visa for all countries, 90-day (2023 Q3+)
- \texttt{covid\_restrict}: Travel restriction index (0-1)

\textbf{Result:} External features improved LR slightly (MAE reduced
\textasciitilde5\%) but made RF/XGB worse due to overfitting.
\texttt{lag\_1} and \texttt{rolling\_mean\_4} remain the dominant
features (\textasciitilde73\% importance). The post-COVID structural
break remains the fundamental challenge.

\section{8. Conclusion}\label{conclusion}

\subsection{8.1 Key Findings}\label{key-findings}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\item
  \textbf{Coverage bias is significant:} Only 11-12 countries reported
  during 2009-2011 vs 30+ from 2012. Raw aggregate totals are
  misleading; growth rates should be used for fair comparison.
\item
  \textbf{China and South Korea dominate} the tourism market, but
  emerging markets (Hong Kong, Spain, Italy, Philippines) grew fastest
  during 2012-2019.
\item
  \textbf{Strong seasonality:} Q1 consistently has the highest arrivals
  (Tet holiday effect).
\item
  \textbf{Linear Regression is surprisingly effective} with only 51
  training samples, achieving the lowest MAE (668K) and best R² (-0.11).
  More complex models overfit.
\item
  \textbf{No model predicts the post-COVID surge well.} All R² values
  are negative. The structural break is invisible to history-based
  models.
\item
  \textbf{CIR\# (state-of-the-art SDE model) fails on this data} due to
  insufficient observations, mean-reversion assumption violation, and
  quarterly granularity. This is a valuable negative finding.
\item
  \textbf{External features (exchange rates, visa policy) provide
  marginal improvement} for LR but not for tree-based models with
  limited data.
\end{enumerate}

\subsection{8.2 Limitations}\label{limitations}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  \textbf{Small training set} (51 quarterly observations after removing
  2021)
\item
  \textbf{2021 gap} breaks time-series continuity
\item
  \textbf{Missing external features:} GDP, flight capacity, Google
  Trends, oil prices
\item
  \textbf{Country coverage inconsistency} (2009-2011 only 11-12
  countries)
\end{enumerate}

\subsection{8.3 Future Directions}\label{future-directions}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  \textbf{Monthly data} to increase sample size and improve seasonal
  capture
\item
  \textbf{External leading indicators:} Source-country GDP, exchange
  rates, Google Trends search volume, flight route capacity
\item
  \textbf{Per-country models} instead of aggregate (different recovery
  trajectories)
\item
  \textbf{Ensemble methods} combining LR (trend) with Chronos (pattern
  recognition)
\item
  \textbf{Regime-switching models} that explicitly handle the pre-COVID
  / during-COVID / post-COVID transitions
\item
  \textbf{Interactive dashboard} for real-time tourism monitoring and
  forecasting
\end{enumerate}
